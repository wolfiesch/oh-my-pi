# Product

## Register

product

## Users

OMP users and maintainers who switch between local profiles, inspect model routing, tune roles, manage provider credentials, and launch companion web tools from a loopback-only Home surface. They are usually in an engineering workflow and need dense, trustworthy state rather than a marketing page.

## Product Purpose

OMP Home is the profile-aware hub for OMP. It exposes the selected profile's routing configuration, agent roster, provider auth state, and companion tool launchers in one local UI. Success means a user can understand where requests route, edit the routing safely, and open the right profile-scoped tool without guessing at CLI flags or paths.

## Brand Personality

Precise, operational, high-agency. The UI should feel like a serious local cockpit: dense enough for experts, readable under pressure, and visually aligned with the shared OMP dark/light token system.

## Anti-references

Do not make this a generic SaaS dashboard, decorative glass panel, gradient-text command center, or card-grid landing page. Avoid ornamental motion, side-stripe accent cards, over-rounded panels, and any design that hides state behind flourish. Do not break parity with the stats, collab-web, and robomp OMP token vocabulary.

## Design Principles

- The routing graph is the hero: make topology legible before making it decorative.
- Trust comes from exact state: profile, process, port, provider auth, and pending edits must be visible and unambiguous.
- Editing stays boring and reversible: existing validation, comment preservation, and mutation paths remain the authority.
- One OMP visual language: reuse the shared OKLCH tokens and component vocabulary instead of inventing a parallel theme.
- Motion explains state changes; it never delays the task.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard access for all controls, reduced-motion alternatives for viewport and reveal motion, visible focus states, and a non-canvas accessibility path for graph nodes through DOM summaries and inspector controls.
